# Actor Metadata
title: YouTube Scraper
description: Crawls and extracts YouTube video, channel, playlist, and hashtag data. Acts as a full YouTube API alternative without quotas. Supports filtering, subtitles, and video attributes.
version: 0.0
build_id: 9Gpb8NaEA6ySAfUte

## Parameters

### searchQueries
type: array[string]
description: Search terms or keywords, same as typing in YouTube's search bar.
behavior: Each query performs a YouTube search and returns results constrained by filters like `sortingOrder`, `dateFilter`, `lengthFilter`, etc.
example: ["ai podcasts", "agentic systems", "n8n tutorials"]

### maxResults
type: integer
minimum: 0
maximum: 999999
default: 0
description: Maximum number of videos to scrape per search term or channel.
behavior: Acts as a per-query video cap. If scraping channels, limits how many regular videos are fetched (not shorts or streams).
example: 100

### maxResultsShorts
type: integer
minimum: 0
maximum: 999999
default: 0
description: Maximum number of Shorts videos to scrape per query.
behavior: Controls how many short-form videos are included. Set to 0 to exclude Shorts entirely.
example: 25

### maxResultStreams
type: integer
minimum: 0
maximum: 999999
default: 0
description: Maximum number of Livestream videos to scrape per query.
behavior: Controls how many streaming or past-stream videos are collected. Set to 0 to skip streams.
example: 10

### startUrls
type: array[object]
description: Direct URLs to scrape (video, channel, playlist, hashtag, or search results page).
behavior: Overrides `searchQueries`. Each object must include a `url` key with a valid YouTube link.
example:
[
  {"url": "https://www.youtube.com/watch?v=xObhZ0Ga7EQ"},
  {"url": "https://www.youtube.com/c/Apify"},
  {"url": "https://www.youtube.com/hashtag/ai"}
]

### downloadSubtitles
type: boolean
description: Enables subtitle download for each scraped video.
behavior: When true, the scraper downloads subtitles and converts them to the format specified in `subtitlesFormat`. Required for `saveSubsToKVS` and `preferAutoGeneratedSubtitles` to function.
example: true

### saveSubsToKVS
type: boolean
description: Saves downloaded subtitles to the key-value store.
behavior: Requires `downloadSubtitles` to be true. Subtitles are stored in `.srt` or chosen format for later retrieval.
example: true

### subtitlesLanguage
type: string
enum: ["any", "en", "de", "es", "fr", "it", "ja", "ko", "nl", "pt", "ru"]
default: "en"
description: Subtitle language preference for subtitle downloads.
behavior: Only applies when `downloadSubtitles` is true. Filters to requested language; `any` downloads available subtitles regardless of language.
example: "en"

### preferAutoGeneratedSubtitles
type: boolean
description: Prefers automatically generated subtitles over user-uploaded ones.
behavior: Only applies if a `subtitlesLanguage` is defined and `downloadSubtitles` is true. Automatically generated captions are prioritized.
example: true

### subtitlesFormat
type: string
enum: ["srt", "vtt", "xml", "plaintext"]
default: "srt"
description: File format for downloaded subtitles.
behavior: Defines subtitle output format stored in the dataset or key-value store.
example: "vtt"

### sortingOrder
type: string
enum: ["relevance", "rating", "date", "views"]
description: Sorts search results according to YouTube sorting options.
behavior:
- `relevance`: Default order matching YouTube search behavior.
- `rating`: Prioritizes highly rated videos.
- `date`: Shows newest videos first.
- `views`: Sorts by total view count.
example: "date"

### dateFilter
type: string
enum: ["hour", "today", "week", "month", "year"]
description: Upload date filter restricting results to a specific timeframe.
behavior: Restricts scraped videos to those uploaded within the selected period. Works only for search-based scraping.
example: "week"

### videoType
type: string
enum: ["video", "movie"]
description: Filters video type for search queries.
behavior: `video` includes standard uploads; `movie` includes full-length movies only.
example: "video"

### lengthFilter
type: string
enum: ["under4", "between420", "plus20"]
description: Duration-based filter.
behavior:
- `under4`: Only videos under 4 minutes.
- `between420`: Between 4 and 20 minutes.
- `plus20`: Longer than 20 minutes.
example: "plus20"

### isHD
type: boolean
description: Filters for HD-quality videos.
behavior: Only retrieves videos available in HD resolution.
example: false

### hasSubtitles
type: boolean
description: Filters for videos that have subtitles or closed captions.
behavior: Limits results to videos with any subtitle availability.
example: true

### hasCC
type: boolean
description: Filters for Creative Commons licensed videos.
behavior: Returns only CC-licensed videos.
example: false

### is3D
type: boolean
description: Filters for 3D-enabled videos.
behavior: Restricts results to 3D content.
example: false

### isLive
type: boolean
description: Filters for live streams.
behavior: Includes only live or upcoming streams in search results.
example: false

### isBought
type: boolean
description: Filters for purchased videos.
behavior: Includes only purchased or premium-access videos.
example: false

### is4K
type: boolean
description: Filters for 4K resolution videos.
behavior: Limits results to videos available in 4K quality.
example: false

### is360
type: boolean
description: Filters for 360° videos.
behavior: Returns spherical immersive videos only.
example: false

### hasLocation
type: boolean
description: Filters for videos with location metadata.
behavior: Restricts results to videos that include geolocation information.
example: false

### isHDR
type: boolean
description: Filters for HDR (High Dynamic Range) videos.
behavior: Includes only HDR-enabled videos.
example: false

### isVR180
type: boolean
description: Filters for VR180 videos.
behavior: Restricts results to stereoscopic 180° VR videos.
example: false

### oldestPostDate
type: string
pattern: "^(\\d{4})-(0[1-9]|1[0-2])-(0[1-9]|[12]\\d|3[01])(T[0-2]\\d:[0-5]\\d(:[0-5]\\d)?(\\.\\d+)?Z?)?$|^(\\d+)\\s*(minute|hour|day|week|month|year)s?$"
description: Minimum upload date for included videos (absolute or relative).
behavior: Filters videos newer than this date or period. Automatically sets sorting to "NEWEST".
example: "7 days"

### sortVideosBy
type: string
enum: ["NEWEST", "POPULAR", "OLDEST"]
description: Sorting order when scraping a channel’s Videos, Shorts, or Live tabs.
behavior:
- `NEWEST`: Scrapes newest uploads first.
- `POPULAR`: Scrapes videos sorted by popularity.
- `OLDEST`: Scrapes oldest uploads first.
example: "POPULAR"

```json
{
  "searchQueries": ["ai tutorials", "rag explanation"],
  "maxResults": 50,
  "maxResultsShorts": 10,
  "maxResultStreams": 5,
  "startUrls": [

    {"url": "https://www.youtube.com/hashtag/ai"}
  ],
  "downloadSubtitles": true,
  "saveSubsToKVS": true,
  "subtitlesLanguage": "en",
  "preferAutoGeneratedSubtitles": true,
  "subtitlesFormat": "vtt",
  "sortingOrder": "date",
  "dateFilter": "month",
  "videoType": "video",
  "lengthFilter": "plus20",
  "isHD": true,
  "hasSubtitles": true,
  "is4K": false,
  "isLive": false,
  "sortVideosBy": "POPULAR"
}
```
